# 🎁 営業先URLまとめ — AIコンサルの案件獲得チャネル一覧

**使い方**:

- 地域チャネルは、地元の商工会議所・商工会や士業団体への接点づくりから。オンラインチャネルは、スポットコンサルやセミナー集客プラットフォームへの登録から始めるのが取り組みやすいです。

- 「登録・登壇できる場」と「相談者が集まる場」は別物なので、両方をバランスよく組み合わせるのがポイントです。

```text

【相手名】様

始めましたのでご報告させてください。

毎回1時間かかっていた作業が10分程度で終わるようになったことでした。
特別なシステム導入ではなく、今お使いのパソコンやツールの中でできる
工夫が中心です。

【相手名】様の会社でも、日々の書類作成や情報整理に時間がかかっている
```

## ② 地域の中小企業向けコールドメール（M365/Google Workspace利用企業向け）

```text

【会社名】

不躾なご連絡で恐縮ですが、1点だけお伝えしたいことがありご連絡いたしました。

実はその契約の中に、すでに生成AIの機能が含まれているケースが多くあります。
新しく何かを契約したり、追加費用を払ったりする前に、
「今の環境の中で使えるAI」を確認するだけで、議事録作成・メール下書き・
資料の要約といった作業時間を大きく減らせることがあります。

たとえば議事録であれば、1時間の会議の要点整理が数分で終わる

【業務名】
```

**カスタマイズのコツ**: 件名と冒頭の「すでにお使いの契約の中に」という一文は削らずに残すと、警戒感より興味が先に立ちやすくなります。

## ⑦ 提案書の骨子（見出し構成と各見出しに書くこと）

**使う場面**: 個別相談やヒアリングを経て、正式な提案書を作成するとき。見出しの骨組みだけを先に固めておくと、内容を埋める作業が早くなります。

```text
提案書 構成テンプレート

1. 表紙
   ・【会社名】御中、提案タイトル、提出日、【自分の名前】/【業務名】を記載

2. はじめに（ヒアリング内容の確認）
   ・打ち合わせで伺った現状の課題を箇条書きで要約
   ・「認識のズレがないか」を先方に確認してもらう位置づけで書く

3. 現状の課題整理
   ・具体的な業務（議事録・資料作成・問い合わせ対応など）ごとに
     かかっている時間や手間を整理
   ・数値化できる部分は「週◯時間」など具体的に記載

4. 提案内容
   ・すでにお使いの環境（M365 / Google Workspace など）の中で
     まず着手できることを最初に提示
```

# 🎁 Claude Code連携術 — AIコンサルの"武器"側セットアップ

顧客に教える生成AIツールはCopilotやGeminiでも、コンサル自身が使う提案書・研修資料・リサーチはClaude Codeで量産する、という役割分担を前提にした連携術です。案件が増えるほど効いてくる「裏方の型」をまとめました。

**受け取り方**: 下のコードブロックをコピーするか、このファイルを保存してください。

## 1. AIコンサル業務用リポジトリのファイル構造テンプレート

```text
ai-consul-ops/
├── CLAUDE.md              # 自分の事業ルール（応答言語・表記規則・提案書の型など）
├── templates/             # 提案書・研修資料・議事録の雛形置き場
├── clients/               # 顧客ごとのフォルダをここに作成
│   └── <顧客名>/
│       ├── hearing.md     # 初回ヒアリングの記録
│       ├── proposal.md    # その顧客向け提案書の実体
│       ├── training/      # 研修スライド・配布資料
│       ├── prompts/       # その顧客向けに調整したプロンプト集
│       └── minutes/       # 打ち合わせ議事録
└── research/              # 業界動向・競合事例などのリサーチ置き場
```

## 2. CLAUDE.md の雛形全文

```text
# AIコンサル業務ルール

- 応答は日本語で行う。専門用語は初出時に一言補足する
- 顧客名・企業名・社外秘情報はファイル内では仮名（A社、B社担当者など）で扱う
- 提案書を書くときは templates/proposal.md の構成順に従う
- 数字・統計・料金は出典が明確な場合のみ記載し、断定できないものは「要確認」と明記する
- 研修資料は専門用語を避け、日常語と比喩で説明する（対象は非エンジニア）
- 顧客の実データ・個人情報・契約書原本はそのまま貼り付けず、要約または仮名化してから扱う
- 議事録は「決定事項」「宿題」「次回日程」の3項目を必ず含める
- 見積もり・料金提示を含む文書は、送付前に必ず人間の確認を挟む
- 同じ顧客向けの資料は clients/<顧客名>/ 配下にまとめ、他顧客のフォルダを参照しない
```

```text
- 最後に「この顧客への提案に使えそうな示唆」を3点まとめる
```

## 4. もう一歩 — 自律実行・分担の使い所

`/goal` はゴールを設定して自律的にタスクを進めさせる機能で、「A社向け提案書一式をtemplates/proposal.mdの型に沿って完成させる」のようなまとまった作業を任せるときに向いています。サブエージェントは調査・リサーチと資料化（スライドや文書への整形）のように性質の異なる作業を並行して分担させる使い方が基本で、たとえば「業界動向の調査」を1つのサブエージェントに、「調査結果を研修スライド構成に落とし込む」作業を別のサブエージェントに任せる、といった分業ができます。案件数が増えて一人で回しきれなくなってきたタイミングで検討するとよい範囲です。まずは3節のプロンプトを手動で使い、必要に応じて自律実行や分担に広げていく順番をおすすめします。

## 5. 注意 — 顧客データの扱い

顧客の実名・連絡先・契約内容・財務データなどの個人情報や社外秘情報は、そのままAIに貼り付けないでください。ヒアリングメモや議事録を扱う際は、CLAUDE.mdのルールに従って仮名化・要約してから入力する運用を徹底してください。見積もりや契約に関わる内容は、AIが作成した下書きであっても送付前に必ず人間が確認してください。

**ポイント**: 顧客への提案は分かりやすいツールで、自分の裏方作業は使い慣れたツールで、という二刀流がコンサル業務の量産効率を左右します。
